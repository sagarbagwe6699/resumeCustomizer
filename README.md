# Resume Customizer - End-to-End Usage Guide

This guide walks you through the complete flow:

1. Use `prompt.md` in Claude.
2. Generate job-specific resume content.
3. Resolve merge-conflict blocks (`<<<<<<<`, `=======`, `>>>>>>>`).
4. Run the generator command to create company-wise output files.

---

## 1) Prerequisites

- Python 3.10+ installed.
- `pandoc` installed and available in your terminal PATH.
- Project files available in this folder:
  - `resume.md` (your base resume)
  - `prompt.md` (the Claude instruction prompt)
  - `template.docx` (DOCX template used for output formatting)
  - `generate_resumes.py` (multi-job parser + generator)

Optional check commands:

```bash
python3 --version
pandoc --version
```

### Install prerequisites (macOS)

If you are on macOS and do not have these tools yet:

1. Install Homebrew (if missing):

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

2. Install Python and Pandoc:

```bash
brew install python pandoc
```

3. (Optional, recommended) Create and activate a virtual environment:

```bash
python3 -m venv .venv
source .venv/bin/activate
python3 -m pip install --upgrade pip
```

4. Verify setup:

```bash
python3 --version
pip3 --version
pandoc --version
```

## 2) Prepare your input

Simple checklist:

- `resume.md` (your base resume)
- 1 or more JDs, each with:
  - title
  - company name
  - full JD text

Use consistent company names (example: always `Cognition`, not `Cognition AI` in one place and `Cognition` in another), because output folders are created from company names.

Quick JD paste format for Claude:

```text
JOB 1
Title: <Role>
Company: <Company>
JD:
<Full JD>
```

Repo examples:

- `resume.md` (base input)
- `prompt.md` (Claude prompt)
- `output/Cognition/resume.md` and `output/Valon/resume.md` (example generated outputs)

---

## 3) Open Claude and paste the prompt

1. Open `prompt.md`.
2. Copy the full prompt text.
3. Open Claude and paste the full prompt as your first message.
4. In the same message, also attach:
   - `resume.md`
   - `list.xlsx` (your JD list file)
5. In that same message, ask Claude to use both files to generate the job-wise optimized output.

Download the .md file that Claude generates

- `optimized_raw.md`

---

## 4) Resolve merge conflicts in Claude output

`generate_resumes.py` expects clean Markdown. So before running the generator, resolve all conflict blocks.
- Open the response generated by Claude and resolve all conflict blocks.
- Each conflict shows Claude's optimized version of your resume content for that specific company.

---

## 5) Run the resume generation command

From project root:

```bash
python3 generate_resumes.py --input non_conflict_optimized_final.md --template template.docx --output-root ./output
```

This will:

- Parse each `# JOB n: ... — <Company>` block
- Extract the resume section from each block
- Create `output/<CompanySlug>/resume.md`
- Generate `output/<CompanySlug>/resume.docx`

### Useful options

- Generate PDF only:

```bash
python3 generate_resumes.py --input optimized_final.md --format pdf
```

- Generate both DOCX and PDF:

```bash
python3 generate_resumes.py --input optimized_final.md --format both
```

- Custom output folder:

```bash
python3 generate_resumes.py --input optimized_final.md --output-root ./my_output
```

- Custom template:

```bash
python3 generate_resumes.py --input optimized_final.md --template ./template.docx
```

---

## 6) Validate generated output

After a successful run, inspect:

- `output/<Company>/resume.md`
- `output/<Company>/resume.docx`
- optionally `output/<Company>/resume.pdf` (if format is `pdf` or `both`)

Open generated files and quickly confirm:

- Company-specific keywords are present.
- Formatting looks correct.
- No leftover conflict markers.
